# 프로젝트 한계점

## 1. 문서 목적

본 문서는 Rhysida 랜섬웨어 분석 및 복호화 툴 제작 프로젝트의 적용 범위와 한계점을 정리하기 위해 작성되었다.

본 프로젝트는 교육 및 연구 목적의 분석 프로젝트이며, 실제 피해 환경의 모든 파일 복구를 보장하는 것을 목표로 하지 않는다.

## 2. 복호화 가능성의 한계

Rhysida 복호화는 암호화에 사용된 key/IV 또는 nonce를 재구성할 수 있는 경우에만 가능하다.

ChaCha20은 올바른 key와 nonce가 없으면 복호화가 어렵기 때문에, 복호화 가능성은 암호 알고리즘 자체가 아니라 key/IV 생성 과정의 분석 가능성에 의존한다.

따라서 다음과 같은 경우 복호화가 어렵다.

- PRNG 시드나 내부 상태를 추정할 수 없는 경우
- 암호화 시점 정보가 남아 있지 않은 경우
- key/IV 생성 방식이 샘플마다 다른 경우
- 파일 메타데이터가 손상되었거나 삭제된 경우
- 동일한 방식으로 keystream을 재현할 수 없는 경우

## 3. PRNG 분석의 한계

PRNG 기반 복호화 분석은 난수 생성기의 내부 상태를 추정할 수 있어야 의미가 있다.

하지만 PRNG가 충분히 안전하게 사용되었거나, 내부 상태 복원에 필요한 정보가 남아 있지 않다면 key/IV 후보를 현실적인 범위로 줄이기 어렵다.

즉, PRNG 분석은 복호화 가능성을 높일 수 있는 방향이지만, 항상 성공하는 방법은 아니다.

## 4. 샘플별 차이

Rhysida 랜섬웨어는 샘플 또는 변종에 따라 구현 방식이 다를 수 있다.

차이가 발생할 수 있는 항목은 다음과 같다.

- 암호화 대상 파일 선정 방식
- key/IV 생성 방식
- 메타데이터 저장 방식
- 파일 확장자 처리 방식
- 랜섬노트 생성 방식
- 자기 삭제 또는 흔적 제거 방식

따라서 하나의 분석 결과가 모든 샘플에 그대로 적용된다고 보기 어렵다.

## 5. 실제 복구도구와의 차이

KISA 등에서 제공하는 실제 복구도구는 검증된 피해 복구를 목적으로 한다.

반면 본 프로젝트의 복호화 툴은 Rhysida 암호화 구조를 이해하고 복호화 가능 조건을 학습하기 위한 분석용 프로토타입이다.

따라서 본 프로젝트의 결과물은 다음과 같은 한계를 가진다.

- 실제 피해 파일 복구 보장 불가
- 모든 Rhysida 변종 대응 불가
- 공식 복구도구 수준의 안정성 보장 불가
- 제한된 조건에서만 복호화 로직 검증 가능

## 6. 동적 분석의 한계

랜섬웨어는 실제 실행 시 파일 암호화, 시스템 변경, 네트워크 통신 등의 위험한 행위를 수행할 수 있다.

따라서 본 프로젝트에서는 악성코드 샘플을 GitHub에 업로드하지 않으며, 실제 실행 분석은 격리된 환경에서만 수행해야 한다.

동적 분석 시 고려해야 할 한계는 다음과 같다.

- 샘플 실행에 따른 시스템 손상 가능성
- 네트워크 전파 위험
- 분석 환경과 실제 피해 환경의 차이
- 샘플 확보 및 검증의 어려움
- 윤리적·법적 제한

## 7. 프로젝트 범위

본 프로젝트의 범위는 다음과 같다.

- Rhysida 랜섬웨어 개요 정리
- 암호화 알고리즘 구조 분석
- PRNG와 ChaCha20 역할 정리
- 정적 분석 및 동적 분석 항목 정리
- IOC 정리
- 복호화 로직 설계
- 제한적인 복호화 프로토타입 구조 설계

본 프로젝트의 범위에 포함하지 않는 항목은 다음과 같다.

- 실제 공격 수행
- 랜섬웨어 제작
- 악성코드 배포
- 실제 피해 시스템 대상 복구 보장
- 공식 복구도구를 대체하는 도구 제작

## 8. 정리

본 프로젝트는 Rhysida 랜섬웨어의 암호화 구조와 복호화 가능 조건을 이해하기 위한 교육용 분석 프로젝트이다.

최종 결과물은 완성형 복구도구가 아니라, 분석 문서와 복호화 로직 설계, 제한적인 프로토타입을 중심으로 구성된다.
