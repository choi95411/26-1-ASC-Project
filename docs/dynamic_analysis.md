팀원이 작성한 복호화 로직은 "암호화 당시의 시간(메타데이터)과 PRNG(난수생성기) 시드값을 기반으로 Key/IV를 역추적하여 복호화한다"는 가설을 담고 있습니다.

이 로직이 실제로 작동하려면, 동적 분석가인 당신이 **Rhysida가 실제로 실행될 때 어떤 타임스탬프를 시드로 쓰는지, 메모리의 어느 타이밍에 Key가 남아있는지**를 실시간으로 모니터링하여 데이터를 제공해야 합니다.

팀원의 복호화 설계에 맞추어, 당신이 수행해야 할 [동적 분석 및 행위 모니터링 수행 로직]을 설계해 드립니다.

# Rhysida 동적 분석 및 실행 행위 모니터링 로직 설계

## 1. 문서 목적

본 문서는 팀의 'Rhysida 복호화 로직'을 지원하기 위해, 격리된 환경에서 Rhysida 랜섬웨어를 동적 구동하고 복호화의 핵심 단서(PRNG 시드, Key/IV 노출 시점, 타임스탬프)를 확보하기 위한 동적 분석 및 모니터링 흐름을 설계하는 것을 목적으로 한다.

---

## 2. 모니터링 기본 방향

복호화 팀이 요구하는 메타데이터와 시드 후보군을 좁히기 위해 다음 3가지 핵심 행위를 집중 관찰한다.

1. **시간 동기화 모니터링:** 랜섬웨어가 암호화 키를 생성할 때 참조하는 시스템 시간(GetSystemTime, GetTickCount 등)의 호출 시점 포착.
2. **메모리 및 API 후킹:** ChaCha20 암호화 키 또는 RSA 공개키가 메모리에 평문으로 로드되는 순간을 포착하여 덤프.
3. **파일 행위 추적:** 파일이 암호화되는 순간의 `$MFT` 타임스탬프 변화 및 암호화 순서(FIFO 등) 기록.

## 4. 단계별 세부 수행 로직

### 5.1 사전 준비 및 환경 격리

* **네트워크 차단:** 가상머신(VM) 환경의 네트워크를 Host-Only로 설정하여 외부 C2 통신 차단 및 내부 전파 방지.
* **시간 동기화:** 분석 장비의 시스템 시간을 기록하고, 네트워크 시간 동기화(NTP) 기능 비활성화.
* **도구 대기:** Process Monitor(필터: Rhysida 프로세스명), x64dbg, Wireshark 실행 및 덤프 준비.

### 5.2 암호화 시드(Seed) 및 API 모니터링

복호화 로직의 '5.3 PRNG 시드 후보 생성'을 지원하기 위한 단계이다.

* **시간 관련 API 추적:** 랜섬웨어가 난수를 생성하기 위해 호출하는 아래 API들을 모니터링하고 호출된 정확한 '값(Value)'과 '타임스탬프'를 기록한다.
* `GetSystemTime`, `GetLocalTime`, `GetTickCount`, `QueryPerformanceCounter`


* **난수 생성 함수 추적:** 암호화 라이브러리(OpenSSL, BCrypt 등) 혹은 C++ 표준 PRNG 함수(`rand`, `srand`, `std::mt19937`) 사용 여부를 디버거로 추적하여 시드값의 가용 범위를 도출한다.

### 5.3 메모리 내 Key/IV 추출 시도

복호화 로직의 **'5.4 key/IV 후보 생성'** 과정을 단축시키거나 검증하기 위한 단계이다.

* **암호화 루틴 일시정지:** 파일 암호화가 시작되는 시점(`WriteFile` 또는 암호화 API 호출 시점)에 디버거를 브레이크포인트(BP)로 잡아 프로세스를 멈춘다.
* **메모리 덤프:** ChaCha20 구조상 메모리에 Key(32바이트)와 Nonce/IV(8~12바이트)가 평문으로 존재하는 구간을 찾아 메모리 덤프를 수행한다.
* **인자값 추출:** 암호화 함수 호출 시 스택(Stack)이나 레지스터에 인자값으로 전달되는 Key 구조체 주소를 추적한다.

### 5.4 실행 행위 및 변조 아티팩트 수집

복호화 로직의 '5.1 파일 목록 수집' 및 '5.2 메타데이터 확인'을 검증하기 위한 단계이다.

* **암호화 메커니즘 확인:** 대용량 파일 분석을 통해 파일 전체를 암호화하는지, 특정 블록만 건너뛰며 암호화하는지(Intermittent Encryption) 행위를 관찰하여 복호화 팀에 암호화 블록 크기를 전달한다.
* **타임스탬프 변조 확인:** 파일 암호화 후 `.rhysida` 확장자로 변경할 때, 의도적으로 타임스탬프를 변조(Timestomping)하는지 Process Monitor를 통해 감시한다. (변조하지 않는다면 파일 수정 시간이 복호화의 강력한 시드 힌트가 됨).
* **방어 무력화 모니터링:** `vssadmin.exe delete shadows` 등 백업 삭제 명령어 실행 여부와 레지스트리 변조 행위를 IOC로 기록한다.

## 5. 분석가로서 복호화 팀에 전달해야 할 최종 결과물

동적 분석 완료 후, 복호화 팀원에게 아래 데이터를 전달하여 복호화 로직을 완성 시킵니다.

1. **Time Logs:** Rhysida가 최초 실행된 시간 및 PRNG 시드로 사용된 API의 타임스탬프 리스트.
2. **Encryption Behavior:** 간헐적 암호화 여부 및 암호화 블록 사이즈 정보 (예: "10MB 이상 파일은 앞부분 1MB만 암호화함").
3. **Memory Dump:** 디버거를 통해 획득한 ChaCha20 Key/IV 추정 메모리 덤프 파일.
4. **IOC Sheet:** 변조된 확장자, 생성된 랜섬노트 파일명, 등록된 레지스트리 경로 정보.
