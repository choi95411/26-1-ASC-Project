# 복호화 로직 설계

## 1. 문서 목적

본 문서는 Rhysida 랜섬웨어 암호화 구조 분석을 바탕으로 복호화 로직의 전체 흐름을 설계하는 것을 목적으로 한다.

본 프로젝트의 복호화 로직은 실제 피해 복구를 보장하는 완성형 복구도구가 아니라, Rhysida 암호화 구조와 복호화 가능 조건을 이해하기 위한 분석용 설계 문서이다.

## 2. 복호화 로직의 기본 방향

Rhysida 복호화는 암호화에 사용된 key/IV 또는 nonce를 재구성할 수 있는 경우에만 가능하다.

따라서 복호화 로직은 다음 방향으로 설계한다.

1. 암호화된 파일을 입력받는다.
2. 파일명, 확장자, 생성 시간 등 메타데이터를 확인한다.
3. PRNG 시드 또는 내부 상태 후보를 생성한다.
4. 후보값을 바탕으로 key/IV 후보를 만든다.
5. ChaCha20 복호화를 시도한다.
6. 복호화 결과가 정상 파일인지 검증한다.
7. 성공한 결과는 별도 경로에 저장한다.
8. 실패한 경우 원본 파일을 보존하고 로그를 남긴다.

## 3. 입력값

복호화 로직에서 사용할 수 있는 입력값은 다음과 같다.

| 입력값 | 설명 |
|---|---|
| 암호화된 파일 | `.rhysida` 등으로 암호화된 파일 |
| 입력 디렉토리 | 복호화 대상 파일이 저장된 경로 |
| 출력 디렉토리 | 복호화 결과를 저장할 별도 경로 |
| 파일 메타데이터 | 파일 크기, 생성 시간, 수정 시간 등 |
| 시드 후보 정보 | PRNG 시드를 추정하기 위한 후보 범위 |
| 파일 형식 정보 | 정상 파일 여부를 확인하기 위한 파일 시그니처 정보 |

## 4. 전체 처리 흐름

복호화 로직의 전체 흐름은 다음과 같다.

```text
[입력 디렉토리]
      |
      v
[암호화 파일 목록 수집]
      |
      v
[파일 메타데이터 확인]
      |
      v
[PRNG 시드 후보 생성]
      |
      v
[key/IV 후보 생성]
      |
      v
[ChaCha20 복호화 시도]
      |
      v
[복호화 결과 검증]
      |
      v
[성공 파일 저장 / 실패 로그 기록]
```

## 5. 단계별 설계

### 5.1 파일 목록 수집

복호화 대상이 되는 파일 목록을 수집한다.

확인할 항목은 다음과 같다.

- 파일명
- 파일 확장자
- 파일 크기
- 파일 경로
- 생성 시간
- 수정 시간

### 5.2 메타데이터 확인

파일 메타데이터는 PRNG 시드 후보를 좁히거나 암호화 시점을 추정하는 데 참고할 수 있다.

확인할 수 있는 메타데이터는 다음과 같다.

- 파일 생성 시간
- 파일 수정 시간
- 암호화된 파일의 크기
- 암호화 전후 확장자 변화
- 동일 시점에 암호화된 파일 묶음

### 5.3 PRNG 시드 후보 생성

PRNG 시드 후보는 암호화 시점, 시스템 시간, 파일 처리 순서 등의 정보를 바탕으로 추정한다.

단, 실제 시드 생성 방식이 확인되지 않은 상태에서는 복호화 성공을 보장할 수 없다.

### 5.4 key/IV 후보 생성

PRNG 시드 후보를 바탕으로 암호화에 사용되었을 가능성이 있는 key/IV 후보를 생성한다.

이 단계는 암호화 알고리즘 분석 결과와 직접적으로 연결된다.

### 5.5 ChaCha20 복호화 시도

생성된 key/IV 후보를 이용해 ChaCha20 복호화를 시도한다.

복호화 시도는 원본 파일을 직접 수정하지 않고, 임시 결과 파일에 대해서만 수행한다.

### 5.6 결과 검증

복호화 결과가 정상 파일인지 검증한다.

검증 기준은 다음과 같다.

- 파일 시그니처 확인
- 확장자와 파일 헤더 일치 여부 확인
- 파일 크기 확인
- 일부 알려진 구조와 비교
- 텍스트 파일의 경우 읽을 수 있는 문자열 비율 확인

## 6. 안전성 고려사항

복호화 로직 설계에서 가장 중요한 점은 원본 파일 보존이다.

따라서 다음 원칙을 적용한다.

- 원본 암호화 파일은 절대 덮어쓰지 않는다.
- 복호화 결과는 별도 출력 디렉토리에 저장한다.
- 실패한 복호화 결과는 구분하여 저장하거나 삭제한다.
- 모든 작업 과정은 로그로 남긴다.
- 일부 파일 복호화 실패가 전체 작업 중단으로 이어지지 않도록 한다.

## 7. 정리

본 복호화 로직은 Rhysida 암호화 구조를 이해하고, PRNG 및 ChaCha20 기반 복호화 가능 조건을 실험적으로 정리하기 위한 설계이다.

실제 복구 가능 여부는 암호화 당시 사용된 key/IV 재구성 가능성, PRNG 내부 상태 추정 가능성, 파일 메타데이터 보존 여부에 따라 달라진다.
