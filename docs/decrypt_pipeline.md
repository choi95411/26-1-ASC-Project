1. Rhysida 암호화 구조 이해
구현에 앞서 Rhysida의 암호화 방식을 정확히 이해하고자 했다. 공개된 기술 분석 자료를 바탕으로 살펴보았을 때, 전체 암호화 흐름은 이렇다. 먼저 srand(time(NULL))로 시스템 시각을 시드로 삼아 LibTomCrypt의 ChaCha20 PRNG를 초기화한다.
이후 파일마다 80바이트의 난수를 생성하는데, 앞 32바이트가 AES-CTR 암호화 키, 다음 16바이트가 IV(초기화 벡터)로 사용된다. 파일 본문은 AES-CTR로 암호화되고, 생성된 키와 IV는 RSA-4096으로 한 번 더 암호화되어 파일 끝에 붙는다.
마지막으로 파일 확장자가 .rhysida로 바뀌고 랜섬노트가 생성된다. 참고로 ChaCha20은 파일을 직접 암호화하는 게 아니라 PRNG(난수 생성기)로만 쓰인다. 실제 파일 암호화는 AES-CTR이다. 취약점의 핵심은 시드에 있다.
Rhysida는 키 생성 시 보안용 난수 생성기(CSPRNG) 대신 C 표준 라이브러리의 rand() 함수를 사용하고, 그 시드가 랜섬웨어 실행 시각(Unix 타임스탬프)이다. 시드는 32비트 정수라 최대 2^32(약 42억) 가지 경우의 수밖에 없다.
암호화된 파일의 수정 시각(mtime)으로 암호화 시점을 추정하면 브루트포스 범위가 수 초 단위로 확 좁혀진다.

2. 구현 계획
총 4개의 Python 모듈로 구성하며, 최종적으로 CLI 형태의 파이프라인으로 통합한다.
seed_generator.py는 파일의 mtime을 입력받아 탐색할 타임스탬프 후보 리스트를 생성한다. 동적 분석 팀이 제공하는 실행 타임스탬프와 교차 검증해 범위를 최소화하는 것이 핵심이다.
prng_reconstruct.py는 LibTomCrypt의 ChaCha20 PRNG 동작을 Python으로 재현한다. 각 시드에 대해 80바이트를 생성하고, 앞 32바이트를 AES 키, 다음 16바이트를 IV로 분리한다. pycryptodome 라이브러리를 활용한다.
decryptor.py는 암호화 파일과 키, IV를 입력받아 AES-CTR 모드로 복호화를 시도한다. 복호화 결과의 파일 헤더(매직 바이트)를 확인해 성공 여부를 판단한다. PDF는 %PDF, PNG는 \x89PNG, DOCX는 PK 헤더 등을 기준으로 삼는다.
pipeline.py는 위 세 모듈을 순서대로 호출하는 메인 스크립트다. "python pipeline.py --file target.rhysida --mtime 1700000000" 형태의 CLI 인터페이스를 제공하고, 성공 시 복호화된 파일을 저장하며 실패 시 범위를 넓혀 재시도하는 로직을 포함한다.

3. 단계별 일정
1단계는 암호화 분석 결과를 수령하고 키 생성 로직을 코드로 옮기는 작업이다.
2단계는 시드 브루트포스와 ChaCha20 PRNG 재현 모듈을 구현한다.
3단계는 복호화 모듈을 구현하고 격리 VM에서 실제 샘플로 검증한다.
4단계는 전체 파이프라인 통합, 예외 처리, 문서화를 마무리한다.

4. 예상 문제점 및 대응 방안
시드 범위가 넓을 경우 연산량이 과다해질 수 있는 건 mtime 정밀도를 높이고 멀티스레딩을 적용해 대응한다.
LibTomCrypt 내부 구현과 Python 재현 결과가 불일치할 수 있는데, 이 경우 LibTomCrypt 소스를 직접 참고하거나 C 바인딩을 검토한다.

5. 개발 환경 및 도구
언어는 Python 3.x를 메인으로 사용하고 필요 시 C/C++를 일부 보조로 쓴다. 주요 라이브러리는 pycryptodome(AES-CTR, ChaCha20)과 argparse(CLI)다.
분석 환경은 Windows 10/11 기반 격리 VM(Host-only 네트워크)이고, 협업은 Git/GitHub과 Notion으로 진행한다.
참고 자료로는 Trend Micro 분석 보고서, KISA 복호화 툴, NCC Group 기술 문서를 활용한다.
