# Rhysida 암호화 알고리즘 분석

## 1. 분석 목적

본 문서는 Rhysida 랜섬웨어의 파일 암호화 구조를 이해하고, 복호화가 가능해지는 조건을 정리하는 것을 목적으로 한다.

Rhysida 복호화 분석의 핵심은 암호 알고리즘 자체를 깨는 것이 아니라, 암호화에 사용된 key/IV 또는 nonce가 어떤 방식으로 생성되는지 파악하는 것이다. 따라서 본 문서에서는 PRNG 기반 키 생성 흐름과 ChaCha20 스트림 암호의 역할을 중심으로 정리한다.

## 2. Rhysida 암호화 흐름 개요

Rhysida 랜섬웨어의 암호화 흐름은 다음과 같이 정리할 수 있다.

1. 암호화 대상 파일 탐색
2. 암호화 대상 파일 선별
3. PRNG 기반 난수 생성
4. 파일 암호화에 사용할 key/IV 또는 nonce 생성
5. ChaCha20 기반 파일 데이터 암호화
6. 암호화된 파일 저장 및 확장자 변경
7. 랜섬노트 생성 또는 관련 흔적 남김

이 중 복호화 가능성과 직접적으로 관련되는 부분은 PRNG 기반 난수 생성 단계와 ChaCha20 암호화 단계이다.

## 3. PRNG의 역할

PRNG는 Pseudo Random Number Generator의 약자로, 난수처럼 보이는 값을 생성하는 알고리즘이다.

Rhysida 분석에서 PRNG가 중요한 이유는 암호화에 사용되는 key/IV 또는 nonce 생성 과정에 관여할 수 있기 때문이다. 만약 PRNG의 시드나 내부 상태를 추정할 수 있다면, 암호화 당시 사용된 key/IV 후보를 재구성할 가능성이 생긴다.

정리하면 PRNG의 역할은 다음과 같다.

- 암호화에 필요한 난수 생성
- key/IV 또는 nonce 생성 과정에 사용
- 암호화 시점, 시스템 상태, 내부 상태 등에 따라 출력값 결정
- 내부 상태를 복원할 수 있다면 복호화 가능성 확보

## 4. ChaCha20의 역할

ChaCha20은 스트림 암호 방식의 대칭키 암호 알고리즘이다.

스트림 암호는 key와 nonce 등을 이용해 keystream을 만들고, 이 keystream을 평문과 XOR하여 암호문을 생성한다. 복호화도 동일한 keystream을 다시 생성한 뒤 암호문과 XOR하는 방식으로 이루어진다.

따라서 ChaCha20 기반 암호화에서 중요한 요소는 다음과 같다.

- 올바른 key
- 올바른 nonce 또는 IV
- 동일한 counter 설정
- 암호화 당시와 동일한 keystream 재생성

즉, ChaCha20 자체를 직접 공격하는 것이 아니라, 암호화 당시 사용된 key/IV를 재구성할 수 있는지가 복호화 가능성의 핵심이다.

## 5. 복호화 가능 조건

Rhysida로 암호화된 파일을 복호화하려면 다음 조건 중 일부가 충족되어야 한다.

- PRNG 시드 또는 내부 상태를 추정할 수 있어야 한다.
- key/IV 생성 방식을 파악할 수 있어야 한다.
- 암호화 시점이나 파일 생성 시간 등 메타데이터가 남아 있어야 한다.
- 암호화 전후 파일 구조를 비교할 수 있어야 한다.
- 일부 파일의 원본 구조나 파일 시그니처를 알고 있어야 한다.
- ChaCha20 keystream을 재현할 수 있어야 한다.

이 조건이 충족되지 않으면 일반적인 방식으로는 복호화가 어렵다.

## 6. 분석 방향

본 프로젝트에서는 다음 항목을 중심으로 암호화 알고리즘을 분석한다.

| 분석 항목 | 설명 |
|---|---|
| PRNG 구조 | 난수 생성 방식과 내부 상태 복원 가능성 확인 |
| key/IV 생성 | 암호화에 사용되는 key와 IV가 생성되는 방식 정리 |

| 메타데이터 활용 | 암호화 시간, 파일 정보 등이 복호화 힌트로 사용될 수 있는지 확인 |
| 복호화 가능 조건 | 어떤 조건에서 복호화가 가능한지 정리 |

## 7. 정리

Rhysida 복호화의 핵심은 ChaCha20 알고리즘 자체를 깨는 것이 아니다. ChaCha20은 올바른 key와 nonce가 없으면 복호화가 어렵기 때문에, 실제 분석의 초점은 암호화 당시 사용된 key/IV 생성 과정을 재구성할 수 있는지에 있다.

따라서 본 프로젝트에서는 PRNG 기반 키 생성 흐름, ChaCha20 사용 구조, 복호화 가능 조건을 중심으로 Rhysida 암호화 알고리즘을 분석한다.
