# Rhysida 암호화 알고리즘 분석

## 1. 분석 대상

- Rhysida Windows PE 변종
- 공개 분석 자료 기준
- 실제 감염 재현이 아니라 방어 목적 분석

## 2. 전체 암호화 흐름

1. 파일 탐색
2. 암호화 대상 선정
3. PRNG 초기화
4. key / IV 생성
5. 파일 암호화
6. 확장자 변경
7. 랜섬노트 생성

## 3. 사용된 암호화 요소

- ChaCha20
- PRNG
- key
- IV 또는 nonce

## 4. 핵심 취약점

Rhysida의 복구 가능성은 암호 알고리즘 자체를 깨는 것이 아니라,
키 생성에 사용된 PRNG 상태를 특정 조건에서 재현할 수 있다는 점에 있다.

## 5. 정리할 내용

- PRNG seed가 어디서 오는지
- 파일별 key/IV가 어떻게 생성되는지
- 파일 순서가 복호화에 영향을 주는지
- 어떤 조건에서 복호화가 가능한지
