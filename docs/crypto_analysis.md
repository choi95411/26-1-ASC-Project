# Rhysida 암호화 알고리즘 분석

## 1. 분석 목적

본 문서는 Rhysida 랜섬웨어의 암호화 구조를 정리하고, 복호화 가능성이 발생한 원인을 암호화 알고리즘 관점에서 분석하는 것을 목적으로 한다.

Rhysida의 복호화 가능성은 암호 알고리즘 자체가 수학적으로 깨졌기 때문이 아니라, 파일 암호화에 필요한 key와 IV를 생성하는 과정에서 사용된 난수 생성기의 내부 상태를 특정 조건에서 재현할 수 있었기 때문에 발생한 것으로 분석된다.

따라서 본 문서에서는 Rhysida의 전체 암호화 흐름, PRNG의 역할, ChaCha20의 역할, AES-256 CTR의 역할, RSA-4096 OAEP의 역할을 구분하여 정리한다.

---

## 2. Rhysida 암호화 구조 개요

공개 분석 자료에 따르면 Rhysida는 파일을 직접 암호화하기 위해 대칭키 암호 방식을 사용한다. 파일마다 암호화에 필요한 key와 IV가 생성되고, 이 값들을 이용해 파일 데이터가 암호화된다.

암호화된 파일을 나중에 복구하지 못하게 만들기 위해, Rhysida는 파일 암호화에 사용한 key와 IV를 공격자의 RSA 공개키로 다시 암호화하여 파일 끝부분에 저장하는 구조를 사용한 것으로 분석된다.

즉, 전체 구조는 다음과 같이 정리할 수 있다.

```text
파일 탐색
    ↓
암호화 대상 파일 선정
    ↓
PRNG를 이용해 파일별 key / IV 생성
    ↓
AES-256 CTR 방식으로 파일 데이터 암호화
    ↓
파일 암호화에 사용한 key / IV를 RSA-4096 OAEP로 암호화
    ↓
암호화된 key / IV 정보를 파일 끝부분에 저장
    ↓
파일 확장자 변경 및 랜섬노트 생성
```

이 구조에서 실제 파일 내용 암호화는 AES-256 CTR이 담당하고, ChaCha20은 파일 자체를 직접 암호화하는 역할보다는 난수 생성기 내부에서 key와 IV를 만들기 위한 난수 생성에 사용되는 역할로 이해하는 것이 적절하다.

---

## 3. 주요 암호화 구성 요소

### 3.1 PRNG의 역할

PRNG는 Pseudo-Random Number Generator의 약자로, 난수처럼 보이는 값을 생성하는 알고리즘이다.

Rhysida에서 PRNG는 파일 암호화에 필요한 key와 IV를 생성하는 핵심 요소이다. 파일을 암호화할 때마다 새로운 key와 IV가 필요하므로, 랜섬웨어는 PRNG를 통해 암호화에 필요한 난수 값을 생성한다.

Rhysida 분석에서 PRNG가 중요한 이유는 다음과 같다.

- 파일 암호화 key가 PRNG 출력값에 의존한다.
- IV 역시 PRNG 출력값에 의존한다.
- PRNG 내부 상태가 재현되면 동일한 난수열을 다시 생성할 수 있다.
- 동일한 난수열을 재생성할 수 있으면 파일 암호화에 사용된 key와 IV 후보를 복원할 수 있다.

따라서 Rhysida 복호화 분석의 핵심은 AES 자체를 깨는 것이 아니라, AES에 들어간 key와 IV가 어떻게 생성되었는지를 추적하는 것이다.

---

### 3.2 ChaCha20의 역할

ChaCha20은 일반적으로 스트림 암호 알고리즘으로 알려져 있지만, Rhysida 분석에서는 파일 데이터를 직접 암호화하는 주 알고리즘이라기보다는 LibTomCrypt에서 제공하는 ChaCha20 기반 PRNG의 구성 요소로 사용된 것으로 정리된다.

즉, Rhysida에서 ChaCha20의 역할은 다음과 같이 볼 수 있다.

```text
ChaCha20 기반 PRNG
    ↓
난수 생성
    ↓
AES key 생성
    ↓
AES IV 생성
    ↓
RSA-OAEP padding 등에 필요한 난수 생성
```

따라서 “Rhysida가 ChaCha20을 사용한다”는 표현은 단순히 “파일을 ChaCha20으로 암호화한다”는 의미로 이해하면 안 된다.

공개 분석 기준으로는 Rhysida가 ChaCha20 기반 PRNG를 이용해 난수를 만들고, 그 난수로 AES-256 CTR 암호화에 필요한 key와 IV를 생성하는 구조로 보는 것이 더 정확하다.

---

### 3.3 AES-256 CTR의 역할

AES는 대칭키 암호 알고리즘이고, Rhysida에서는 파일 데이터를 암호화하는 데 사용된 것으로 분석된다.

AES-256은 256비트 길이의 key를 사용하는 AES 방식이다. CTR 모드는 Counter Mode의 약자로, 블록 암호인 AES를 스트림 암호처럼 사용할 수 있게 하는 운용 모드이다.

AES-256 CTR의 특징은 다음과 같다.

- 같은 key와 IV 또는 nonce가 주어지면 동일한 keystream이 생성된다.
- 암호화와 복호화가 구조적으로 단순하다.
- 병렬 처리에 유리하여 대용량 파일 암호화에 적합하다.
- key와 IV가 안전하게 생성되고 재사용되지 않는다는 전제가 매우 중요하다.

Rhysida는 암호화 속도를 높이기 위해 파일 전체를 항상 모두 암호화하지 않고, 파일 크기에 따라 일부 구간만 암호화하는 방식도 사용한 것으로 분석된다. 이는 랜섬웨어가 짧은 시간 안에 많은 파일을 손상시키기 위한 성능 최적화로 볼 수 있다.

---

### 3.4 RSA-4096 OAEP의 역할

RSA는 공개키 암호 알고리즘이다. 대칭키 암호인 AES와 달리, 공개키와 개인키를 분리해서 사용한다.

Rhysida 구조에서는 AES key와 IV를 이용해 파일을 암호화한 뒤, 그 AES key와 IV를 공격자의 RSA 공개키로 암호화하여 파일 끝부분에 저장하는 방식이 사용된 것으로 분석된다.

이 구조의 목적은 다음과 같다.

```text
파일 데이터
    → AES key / IV로 빠르게 암호화

AES key / IV
    → RSA 공개키로 암호화하여 보호

RSA 개인키
    → 공격자만 보유
```

즉, 피해자가 암호화된 파일을 가지고 있어도 RSA 개인키가 없으면 파일 암호화에 사용된 AES key와 IV를 직접 복원하기 어렵게 만드는 구조이다.

OAEP는 RSA 암호화에 사용되는 padding 방식이다. RSA를 그대로 사용하는 것보다 안전성을 높이기 위해 사용된다.

---

## 4. 전체 암호화 흐름 상세 정리

### 4.1 파일 탐색 및 대상 선정

Rhysida는 시스템 내 파일을 탐색하고 암호화 대상 파일을 선정한다. 일반적으로 운영체제 동작에 필수적인 시스템 디렉토리나 특정 파일은 제외된다.

이러한 제외 처리는 랜섬웨어가 시스템을 완전히 부팅 불가능하게 만드는 것보다, 사용자가 시스템에 접근할 수는 있지만 중요한 파일은 열 수 없게 만드는 방향으로 동작하기 위한 것으로 볼 수 있다.

---

### 4.2 파일별 key와 IV 생성

암호화 대상 파일이 정해지면, 각 파일에 대해 AES 암호화에 사용할 key와 IV가 필요하다.

이때 Rhysida는 LibTomCrypt의 ChaCha20 기반 PRNG를 이용해 난수를 생성하고, 이 난수로 파일별 AES key와 IV를 만든 것으로 분석된다.

이 부분이 Rhysida 복호화 분석에서 가장 중요한 지점이다.

```text
PRNG 내부 상태
    ↓
난수 출력
    ↓
AES-256 key 생성
    ↓
AES CTR IV 생성
```

PRNG가 안전하게 초기화되고 충분한 entropy를 사용한다면, 동일한 key와 IV를 다시 추측하는 것은 현실적으로 어렵다.

하지만 구현상 취약점으로 인해 감염 당시의 PRNG 내부 상태를 재현할 수 있다면, 동일한 난수열을 다시 만들 수 있고, 그 결과 파일 암호화에 사용된 key와 IV 후보도 재생성할 수 있다.

---

### 4.3 AES-256 CTR 파일 암호화

파일별 key와 IV가 준비되면 Rhysida는 AES-256 CTR 방식으로 파일 데이터를 암호화한다.

CTR 모드는 counter 값을 증가시키며 keystream을 만들고, 이 keystream을 원본 데이터와 결합하여 암호문을 만든다.

개념적으로는 다음과 같다.

```text
AES(key, counter/IV) → keystream 생성
plaintext XOR keystream → ciphertext
```

복호화도 같은 keystream을 다시 만들어 ciphertext와 결합하면 plaintext가 복원된다.

```text
ciphertext XOR 같은 keystream → plaintext
```

따라서 AES-CTR에서는 key와 IV가 정확히 복원되는 것이 매우 중요하다. key나 IV가 하나라도 다르면 keystream이 달라지고, 복호화 결과도 정상 파일이 되지 않는다.

---

### 4.4 key와 IV의 RSA 암호화

파일 암호화에 사용된 AES key와 IV는 그대로 저장되지 않고, RSA-4096 OAEP 방식으로 암호화되어 파일 끝부분에 저장된 것으로 분석된다.

이 구조는 일반적인 랜섬웨어에서 자주 사용되는 hybrid encryption 구조와 유사하다.

```text
파일 내용 → AES로 암호화
AES key/IV → RSA 공개키로 암호화
RSA 개인키 → 공격자만 보유
```

이렇게 하면 AES는 빠른 파일 암호화를 담당하고, RSA는 AES key와 IV를 보호하는 역할을 담당한다.

---

## 5. 복호화 가능성이 생긴 이유

Rhysida 복호화 가능성의 핵심은 AES-256 CTR이나 RSA-4096 OAEP 자체가 깨졌다는 뜻이 아니다.

공개 연구에서 설명하는 핵심은 다음과 같다.

```text
암호 알고리즘 자체의 취약점
    X

PRNG 구현 또는 초기화 과정의 취약점
    O
```

즉, 암호화에 사용된 key와 IV가 안전한 방식으로 완전히 예측 불가능하게 생성되었다면 복호화는 매우 어렵다. 그러나 PRNG의 내부 상태를 감염 당시 기준으로 재현할 수 있다면, Rhysida가 생성했던 난수열을 다시 만들 수 있다.

그 결과 다음 흐름이 가능해진다.

```text
PRNG 내부 상태 재현
    ↓
동일한 난수열 생성
    ↓
파일별 AES key / IV 후보 생성
    ↓
AES-CTR 복호화 시도
    ↓
정상 파일 구조 확인
```

따라서 Rhysida 복호화 분석의 핵심은 암호문을 강제로 해독하는 것이 아니라, 암호화 당시 사용된 난수 생성 과정을 되돌려 key와 IV를 재생성하는 것이다.

---

## 6. PRNG와 복호화 로직의 관계

Rhysida에서 PRNG는 단순한 보조 기능이 아니라 복호화 가능성을 결정하는 핵심 요소이다.

랜섬웨어의 파일 암호화는 다음과 같은 의존 관계를 가진다.

```text
PRNG 상태
    ↓
난수 출력
    ↓
AES key / IV
    ↓
AES-CTR keystream
    ↓
암호문 생성
```

복호화 관점에서는 이 흐름을 반대로 생각할 수 있다.

```text
암호화된 파일
    ↓
정상 파일 구조 검증
    ↑
AES-CTR 복호화 결과
    ↑
AES key / IV 후보
    ↑
PRNG 출력 재생성
    ↑
PRNG 상태 재현
```

즉, 복호화 툴은 단순히 암호화된 파일을 입력받아 바로 복구하는 것이 아니라, 가능한 PRNG 상태에서 생성되는 key/IV 후보를 이용해 복호화를 시도하고, 그 결과가 정상 파일인지 검증하는 구조를 가진다.

---

## 7. ChaCha20 자체와 ChaCha20 PRNG의 구분

Rhysida 분석에서 혼동하기 쉬운 부분은 ChaCha20의 역할이다.

ChaCha20은 원래 스트림 암호 알고리즘이지만, Rhysida에서는 공개 분석 기준으로 파일 본문 암호화의 직접 알고리즘이라기보다 ChaCha20 기반 PRNG의 형태로 사용된 것으로 정리된다.

따라서 역할을 구분하면 다음과 같다.

| 구성 요소 | Rhysida 내 역할 |
|---|---|
| ChaCha20 기반 PRNG | AES key, IV, RSA-OAEP padding 등에 필요한 난수 생성 |
| AES-256 CTR | 실제 파일 데이터 암호화 |
| RSA-4096 OAEP | 파일 암호화에 사용된 AES key와 IV 보호 |
| PRNG 내부 상태 | key/IV 재생성 가능성과 직접 연결되는 핵심 분석 대상 |

이 구분이 중요하다. 만약 ChaCha20을 파일 암호화 알고리즘으로 잘못 이해하면, 실제 복호화 로직 설계 방향도 잘못 잡힐 수 있다.

---

## 8. 암호화 방식의 의미

Rhysida는 빠른 암호화를 위해 대칭키 암호를 사용하고, 대칭키를 보호하기 위해 공개키 암호를 사용하는 hybrid encryption 구조를 사용한다.

이 방식 자체는 보안적으로 일반적인 구조이다. 문제는 구조 자체가 아니라, 파일별 key와 IV를 생성하는 난수 생성 과정에서 구현 취약점이 존재했다는 점이다.

정리하면 다음과 같다.

```text
AES-256 CTR 사용
    → 파일 암호화 속도를 높이기 위한 선택

RSA-4096 OAEP 사용
    → AES key와 IV를 공격자 공개키로 보호하기 위한 선택

ChaCha20 PRNG 사용
    → 암호화에 필요한 난수 생성 목적

복호화 가능성
    → PRNG 내부 상태 재현 가능성에서 발생
```

따라서 Rhysida 분석의 핵심은 “어떤 암호 알고리즘을 사용했는가”보다 “그 알고리즘에 들어가는 key와 IV가 어떻게 만들어졌는가”에 있다.

---

## 9. 분석 요약

Rhysida 랜섬웨어의 암호화 구조는 대칭키 암호와 공개키 암호를 결합한 hybrid encryption 구조로 볼 수 있다.

파일 데이터는 AES-256 CTR 방식으로 암호화되고, 파일 암호화에 사용된 AES key와 IV는 RSA-4096 OAEP 방식으로 암호화되어 파일에 저장된다. 이때 AES key와 IV를 생성하기 위해 LibTomCrypt의 ChaCha20 기반 PRNG가 사용된 것으로 공개 분석에서 설명된다.

복호화 가능성은 AES-256이나 RSA-4096 자체를 깨는 방식에서 나온 것이 아니다. 핵심은 PRNG의 내부 상태를 감염 당시 기준으로 재현할 수 있었고, 이를 통해 동일한 난수열과 key/IV 후보를 다시 만들 수 있었다는 점이다.

따라서 Rhysida 복호화 로직 설계에서 가장 중요한 분석 대상은 다음과 같다.

- PRNG 내부 상태
- PRNG 초기화 방식

## 10. 결론

Rhysida의 암호화 구조는 겉으로 보면 강한 암호 알고리즘을 조합한 구조이다. AES-256 CTR, RSA-4096 OAEP, ChaCha20 기반 PRNG는 각각 독립적으로 보면 널리 사용되는 암호 기술이다.

그러나 랜섬웨어 복호화 가능성은 암호 알고리즘의 이름만으로 결정되지 않는다. 실제 구현에서 key와 IV가 어떻게 생성되고, PRNG가 어떻게 초기화되며, 내부 상태가 외부 조건으로부터 재현 가능한지가 더 중요할 수 있다.

Rhysida 사례는 안전한 암호 알고리즘을 사용하더라도 난수 생성 과정의 구현이 부적절하면 전체 암호화 구조가 약화될 수 있음을 보여준다.

본 프로젝트에서는 이 점에 초점을 맞추어 Rhysida의 PRNG 기반 key/IV 생성 흐름을 분석하고, 이를 바탕으로 방어 목적의 복호화 로직 설계로 확장한다.
