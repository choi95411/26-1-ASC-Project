# [제 2차 프로젝트 진행 보고서] Rhysida 랜섬웨어 분석 및 복호화 툴 제작

## 팀 전체 진행 현황

- **이번 회차 목표:**  
  Rhysida 랜섬웨어의 암호화 구조, 정적 분석 방향, 동적 분석 환경, 복호화 툴 설계 방향을 구체화하고 각 담당 분야별 분석 문서 초안을 작성한다.

- **현재 진행률:** 30% (전체 일정 대비)

- **주요 달성 사항:**  
  - Rhysida 랜섬웨어의 암호화 흐름과 복호화 가능 조건을 정리하였다.
  - PRNG 기반 key/IV 생성 방식과 ChaCha20 스트림 암호 사용 구조를 중심으로 암호화 알고리즘 분석 방향을 구체화하였다.
  - 정적 분석 담당자는 바이너리 섹션 구조, IAT, 주요 API 호출 지점, 제어 흐름 분석 방법을 학습하였다.
  - 동적 분석 담당자는 가상 환경 구성 방식, 네트워크 모드, 실행 모니터링 항목, 메모리 구조, API 후킹 개념을 학습하였다.
  - 복호화 툴 개발 담당자는 복호화 파이프라인의 전체 흐름과 안전한 파일 처리 방식을 정리하였다.
  - `crypto_analysis.md`, `decrypt_logic_design.md`, `limitations.md`, `static_analysis.md`, `dynamic_analysis.md` 문서를 중심으로 역할별 문서화 작업을 진행하였다.

## 이슈 및 해결 방안

- **문제 상황:**  
  - Rhysida 랜섬웨어의 암호화 구조와 복호화 가능 조건을 이해하기 위해 PRNG, key/IV 생성 방식, ChaCha20 스트림 암호 구조를 함께 파악해야 했다.
  - 정적 분석과 동적 분석에서 확인해야 할 항목이 많아, 각 담당자의 분석 범위를 명확히 나눌 필요가 있었다.
  - 실제 악성코드 샘플을 직접 다루는 경우 보안상 위험이 있으므로, GitHub에 업로드 가능한 자료와 로컬 격리 환경에서만 관리해야 하는 자료를 구분해야 했다.
  - 복호화 툴은 실제 복구를 보장하기 어렵기 때문에, 프로젝트 결과물의 범위를 분석용 프로토타입 수준으로 명확히 정리할 필요가 있었다.

- **해결 현황:**  
  - 암호화 분석은 PRNG 역할, key/IV 생성, ChaCha20 사용 구조, 복호화 가능 조건으로 범위를 나누어 정리하였다.
  - 정적 분석은 바이너리 섹션 구조, IAT, 주요 API 호출, 제어 흐름 분석을 중심으로 진행하기로 하였다.
  - 동적 분석은 가상 환경 구성, 실행 모니터링 항목, 파일·프로세스·네트워크 행위 관찰을 중심으로 진행하기로 하였다.
  - 악성코드 샘플, 실행 파일, 암호화된 파일 등은 GitHub에 업로드하지 않고, 문서와 분석 결과만 저장하기로 하였다.
  - 복호화 툴은 실제 피해 복구용 도구가 아니라, 암호화 구조와 복호화 가능 조건을 이해하기 위한 교육용 프로토타입으로 범위를 정하였다.
