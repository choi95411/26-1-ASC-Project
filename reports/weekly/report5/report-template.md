# [제 5차 프로젝트 진행 보고서] Rhysida 랜섬웨어 복호화 분석 프로젝트

- **팀원:** (팀장) 20221525_최민혁, (팀원) 20251739_이지원, 학번_안시후, 20243303_최지우
- **활동 기간:** 2026. 07. 08. ~ 2026. 07. 21. (2주)

## 팀 전체 진행 현황

- **이번 회차 목표:** 4차에서 정리한 암호화 구조와 복호화 설계를 바탕으로, 안전한 dummy 파일 기반 테스트 암호화 코드, 파일 구조 parser, 복호화 demo를 연결하여 복호화 파이프라인의 동작 가능성을 검증한다. 동시에 정적 분석, 동적 분석, 테스트 환경 정리 결과를 5차 보고서에 반영하여 팀 전체 산출물의 연결성을 높인다.
- **현재 진행률:** 70% (전체 일정 대비)
- **주요 달성 사항:**
  - 팀장/복호화 구현 파트에서는 `src/safe_test_encrypt.py`, `src/rhysida_file_parser.py`, `src/recovery_demo.py`를 연결하여 dummy 파일 기준 암호화 -> parser 분석 -> AES-CTR 복호화 -> SHA-256 검증 흐름을 확인하였다.
  - 테스트용 footer metadata에 AES key, IV, 원본 SHA-256, 암호화 본문 SHA-256, 원본 파일명을 저장하는 구조를 구현하고, `tests/test_vectors/expected_result.md`에 5차 검증 결과를 정리하였다.
  - 정적 분석 파트에서는 바이너리 내 대칭키 생성 함수, RSA 공개키 연동 지점, PRNG 시드 관련 흐름, 간헐적 암호화 구조를 분석하여 복호화 가능 조건과 파일 구조 분석의 근거를 보강하였다.
  - 테스트 환경/시연 파트에서는 dummy 테스트 벡터와 실행 순서를 기준으로 안전한 검증 흐름을 정리하고, 실제 악성 샘플이나 피해 파일을 사용하지 않는 범위를 유지하였다.
  - 동적 분석/모니터링 파트에서는 향후 실행 행위 검증에 필요한 관찰 항목을 정리하고, 파일 변경, 확장자 변경, footer 생성 여부, 네트워크 행위 여부를 확인할 기준을 보완하는 방향으로 진행하였다.
  - 팀 전체적으로는 문서 분석 결과와 코드 검증 결과를 연결하여 6차 보고서에서 최종 산출물 수준으로 정리할 기반을 마련하였다.

## 이슈 및 해결 방안

- **문제 상황:** 이번 차수에서는 dummy 파일 기반 파이프라인은 연결되었지만, 현재 구현은 테스트용 footer에 AES key/IV를 저장하는 안전한 검증 구조이다. 따라서 실제 피해 파일 복구 가능성을 보장하는 단계는 아니며, 실제 Rhysida footer 구조와 동일하다고 단정할 수 없다. 또한 정적 분석, 동적 분석, parser/demo 검증 결과가 아직 하나의 최종 보고서 흐름으로 완전히 통합되지는 않았다.
- **해결 현황:** 프로젝트 범위를 교육용/분석용으로 제한하고, 단일 dummy 파일만 처리하도록 구현하였다. 정적 분석 파트는 복호화 가능 조건과 파일 구조 분석 근거를 보강하고, 구현 파트는 SHA-256 비교로 복구 성공 여부를 검증하도록 구성하였다. 동적 분석 파트는 6차에서 실행 행위 관찰 결과와 IOC 정리를 보완할 예정이다. 다음 차수에서는 각 팀원 산출물을 최종 보고서와 발표 자료 흐름에 맞게 통합한다.

## 다음 회차 목표

- 구현 파트는 `safe_test_encrypt.py`, `rhysida_file_parser.py`, `recovery_demo.py`의 오류 처리와 예외 메시지를 보강한다.
- 테스트 파트는 footer 손상, key/IV 불일치, 잘못된 입력 파일 등 실패 케이스를 추가 테스트한다.
- parser 출력 JSON을 사용하는 복호화 실행 흐름을 추가 검증한다.
- 정적 분석 파트는 파일 구조, 간헐적 암호화, key/IV 생성 조건 분석 결과를 최종 문서에 맞게 정리한다.
- 동적 분석 파트는 격리 환경 실행 관찰 결과, 파일 변경 행위, 확장자 변경, footer 생성 여부, 네트워크 행위 여부를 확인하고 IOC 문서와 연결한다.
- `ioc.md`, `limitations.md`, 동적 분석 문서와 코드 검증 결과를 연결한다.
- 최종 보고서와 발표 자료에 사용할 전체 파이프라인 도식, 실행 명령, SHA-256 검증 결과를 정리한다.
- 6차 보고서에는 팀원별 분석 결과와 최종 산출물 수준의 코드 구조, 통합 테스트 결과를 함께 반영한다.
