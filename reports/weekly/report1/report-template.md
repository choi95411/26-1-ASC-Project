## 이슈 및 해결 방안

- **문제 상황:**  
  - 팀원들이 Git 사용 방식에 익숙하지 않아 협업 규칙을 단순하게 정할 필요가 있었다.
  - 각 담당 분야별 분석 방향은 정해졌으나, 아직 본격적인 실습과 문서화는 초기 단계이다.

- **해결 현황:**  
  - 초기에는 복잡한 branch 전략을 사용하지 않고, 팀원별 담당 파일을 나누어 작업하기로 하였다.
  - 작업 전 `git pull`, 작업 후 `git add`, `git commit`, `git push` 순서를 기본 협업 규칙으로 정리하였다.
  - 각 팀원은 다음 회차부터 담당 분야의 학습 내용을 바탕으로 분석 문서와 보고서를 작성하기로 하였다.
