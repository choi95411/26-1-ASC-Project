## 이슈 및 해결 방안

- **문제 상황:** 
  - GitHub에서 빈 디렉토리는 자동으로 표시되지 않아 `src`, `tests`, `reports/weekly/report1` 등의 폴더가 보이지 않는 문제가 있었다.
  - Git branch, commit, push 등의 Git 사용 방식이 익숙하지 않아 초기 협업 구조를 단순하게 구성할 필요가 있었다.
  - 랜섬웨어 샘플을 다루는 프로젝트 특성상 GitHub에 업로드하면 안 되는 파일의 범위를 명확히 정해야 했다.

- **해결 현황:** 
  - 빈 디렉토리가 GitHub에 보이도록 각 폴더에 `.gitkeep` 파일을 추가하는 방식으로 해결하였다.
  - 초기 단계에서는 복잡한 branch 전략을 사용하지 않고, 팀원별 담당 파일을 나누어 수정하는 방식으로 진행하기로 하였다.
  - `.gitignore`에 `*.exe`, `*.dll`, `*.bin`, `*.zip`, `*.7z`, `*.rhysida`, `*.pcap`, `*.dmp` 등 위험하거나 대용량인 파일 형식을 추가하여 업로드를 방지하였다.
  - 작업 전 `git pull`, 작업 후 `git add`, `git commit`, `git push` 순서를 지키는 기본 규칙을 팀 협업 방식으로 정리하였다.
