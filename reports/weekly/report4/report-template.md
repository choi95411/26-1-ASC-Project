- **이번 회차 목표:** Rhysida 암호화 구조 분석 내용을 바탕으로 안전한 dummy 파일 기반 복호화 파이프라인의 초안을 구성하고, 정적 분석 진행 상황과 테스트 검증 기준을 4차 보고서 형태로 정리한다.

## 이슈 및 해결 방안

- **문제 상황:** 복호화 파이프라인은 `safe_test_encrypt.py`, `rhysida_file_parser.py`, `recovery_demo.py`가 함께 연결되어야 전체 흐름을 검증할 수 있다. 그러나 이번 차수에서는 테스트 암호화 코드와 parser 코드가 아직 전달되지 않아 end-to-end 통합 검증까지는 완료하지 못하였다.
- **해결 현황:** 우선 복호화 담당 모듈인 `recovery_demo.py`를 독립적으로 작성하고, 테스트용 footer 규격에 맞춘 dummy `.rhysida` 파일을 생성하여 AES-CTR 복호화와 SHA-256 검증 흐름을 단독으로 확인하였다. 통합 검증은 다음 차수에서 팀원 코드가 준비된 뒤 진행한다.

## 다음 회차 목표

- `safe_test_encrypt.py`를 테스트 footer 규격에 맞게 구현 또는 수령한다.
- `rhysida_file_parser.py`가 암호화 본문과 footer metadata를 올바르게 분리하는지 확인한다.
- parser 출력과 `recovery_demo.py` 입력 형식을 맞춘다.
- dummy 파일 기준으로 암호화 → parser 분리 → AES-CTR 복호화 → SHA-256 검증까지 전체 파이프라인을 실행한다.
- 통합 테스트 결과를 5차 보고서에 반영한다.
- 복구 가능 조건과 한계를 테스트 결과와 함께 정리한다.
