## 팀 전체 진행 현황

- **이번 회차 목표:** Rhysida 랜섬웨어의 암호화 구조와 실행 흐름을 정적 분석, 동적 분석, 암호화 알고리즘 분석 관점에서 병행 검증한다. 정적 분석에서는 감염 제외 대상 문자열 비교 조건과 메인 암호화 루프의 블록 크기 계산 방식을 확인하고, 동적 분석에서는 x64dbg 기반 디버깅 및 메모리 분석 방법을 학습하여 암호화 함수 호출 시점의 레지스터, 스택, 메모리 인자 추적 방법을 정리한다. 암호화 알고리즘 분석에서는 ChaCha20 PRNG, AES-CTR, RSA-4096의 역할을 구분하고, 이를 바탕으로 복호화 재현 파이프라인의 설계 방향을 문서화한다.
- **현재 진행률:** 35% (전체 일정 대비)
- **주요 달성 사항:** Rhysida 암호화 구조에서 ChaCha20이 파일 본문을 직접 암호화하는 것이 아니라 PRNG로 사용되어 AES-CTR key와 IV를 생성한다는 점을 정리했고, 실제 파일 본문 암호화는 AES-CTR로 수행되며 AES key/IV는 RSA-4096으로 보호되어 파일 끝에 append되는 구조로 문서 방향을 수정했다. 정적 분석 측면에서는 감염 제외 대상 문자열 비교 조건과 메인 암호화 루프 내부의 블록 크기 계산 공식을 추가로 추적했으며, 파일 I/O 관련 API 호출 지점과 레지스터 변화 상태를 점검했다. 동적 분석 측면에서는 x64dbg의 기본 사용법, x86-64 레지스터 및 스택 프레임 구조, WriteFile 등 핵심 API 호출 시점의 브레이크포인트 설정 방법, 암호화 직전 메모리 덤프를 통한 Key/IV 후보 확보 방법을 학습했다. 이를 바탕으로 암호화 알고리즘 분석 문서, 복호화 로직 설계 문서, 동적 분석 문서의 방향성을 정리했다.

## 이슈 및 해결 방안

- **문제 상황:** 기존 분석 문서 일부에서 ChaCha20의 역할이 파일 본문 암호화로 표현되어 있어, 실제 암호화 구조와 문서 내용 사이에 불일치가 발생했다. 
- **해결 현황:** ChaCha20을 파일 본문 암호화가 아닌 PRNG 역할로 정정하고, AES-CTR이 실제 파일 본문 암호화를 담당하며 RSA-4096이 AES key/IV를 보호하는 구조로 문서 방향을 수정했다. 정적 분석에서 확인한 감염 제외 조건 및 암호화 루프 분석 내용과 동적 분석에서 학습한 디버깅·메모리 추적 방법을 암호화 알고리즘 분석 및 복호화 파이프라인 설계와 연결하는 방향으로 정리했다.
